# v1.1 (진단 프루닝 + 전략 세션 문답 + 보고 게이트) 레드팀 기록

작성 원칙: 구멍은 실행 회차가 아니라 작성 시점 자기심문으로 잡는다. v1.0 레드팀(redteam-v1.0.md)의 후속.

## 전환 요약

- 진단 프루닝: 수치를 결정 입력(실측 의무 불변)과 위반 증거(재설계 회차에서 표본 2~3 캡)로 분리. 과감도가 최상위 프루너. 전수 실측의 유일한 집은 회귀.
- 문답의 전략 세션화: 전략 프레임 승인 → 축 캐스케이드 → 파생 좁히기 3단. 질문 4요소 서식(위치 태그 + 상류 확정 인용 + 선택지·추천 + 이 답이 결정하는 것).
- 보고 게이트: 계층 진단 보고서(고정 규격) 제시 전 질문 금지. 계층 완료 보고서 신설. 점검만(brief-all) 경로는 브리프 전문 verbatim.
- 고정 명칭: 1 기획에 따른 콘텐츠 구조화 / 2 레이아웃 최적화 / 3 타이포 위계 정리·불필요한 텍스트 제거 / 4 컬러 스킴 최적화.

## 악용 경로와 차단 규칙

| # | 신규 규칙 | 악용/붕괴 경로 | 차단 |
|---|---|---|---|
| 1 | 위반 증거 표본화 | 최선 표본을 골라 축을 통과시킴 (worst-cluster 규칙이 막던 알려진 loophole 의 재입) | 표본은 **위반 예시 전용, 통과 증거 사용 금지**. 통과 판정이 필요한 축 = 현행 유지 가지 = 전수 유지. 재선언 가지에서는 통과 판정 자체가 불필요하므로 loophole 미성립 |
| 2 | 과감도 프루닝 | "표면 정리만" 회차에도 표본으로 때움 - 그 회차는 수정이 항목 단위라 개별 이탈이 곧 수정 목록 | 프루닝은 과감도 "전면 재설계 허용/구조 조정까지" 회차 전용. "표면 정리만" 또는 stage0 부재 = 전수 유지 |
| 3 | 수치 2종 분리 | 결정 입력 수치까지 표본으로 강등해 축 판정이 눈대중으로 회귀 | 결정 입력 목록을 계층별 폐쇄 열거로 박음 (1: 오퍼 수 N·전환 목표 신호 / 2: 분할선 x·컨테이너 전략·간격 base 후보 / 3: 역할별 font-size·base 후보 / 4: hue 수 vs 역할 수). 이 목록은 실측 의무 불변 |
| 4 | 전략 프레임 | 요식화: 한 줄 걸치고 축 질문 직행 - "전략 세션"이 이름만 남음 | 프레임 필수 3요소 (문제의 구조 + 추천 전략 + 그 전략을 구성하는 결정 목록) 를 진단 보고서의 마지막 절로 형식 강제. 요소 결손 = 보고서 무효 = 문답 개시 불가 |
| 5 | 계층 진단 보고서 | 열화 요약: SSOT id·file:line 을 떨군 매끈한 산문으로 대체 (증거 유실의 사용자 대면 판) | 보고서에 SSOT id·file:line 보존 의무. 브리프의 결정 축 표·문제 증거를 구조 그대로 옮긴다 - 재서술은 허용, 증거 탈락은 무효 |
| 6 | 계층 완료 보고서 | 회귀 분식: 미달 항목을 완료 보고서의 서사로 가림 | 회귀 결과는 미달 = "미완료" 표기 그대로 노출 의무. 보고서가 회귀 결과와 모순되면 보고서 무효 (수치 뒤에 숨지 말 것의 보고서 판) |

## 유지 확인 (전환으로 건드리면 안 되는 것)

- strict 모드는 프루닝 대상이 아니다: 무인 검증은 사람이 없어 표본 신뢰 불가 - 전수 유지.
- 회귀 전수 (래더 밖 = 0, 업스케일 = 0 게이트), payload 전수 계약, 질문 캡 (전략 승인 1 + 축 3~4 + 파생 ≤2, 라운드 최대 2회).
- 게슈탈트 카피 전독 (카피 텔 이중 가중치의 근간 - 카피는 전수 읽기가 곧 판정이라 프루닝 불가).
- v1.0 구조 전체: 4계층·stage 규격·목차 트리 계약·상류 개정 루프·심문 캡.
