# v1.0 (4계층 튜토리얼 전환) 레드팀 기록

작성 원칙: 구멍은 실행 회차가 아니라 작성 시점 자기심문으로 잡는다. 신규 규칙마다 악용 경로를 먼저 죽이고 본문을 쓴다. 이 표는 다음 개정 때 재사용한다.

## 전환 요약

- 계층 재편: `1 콘텐츠 → 2 레이아웃(+타이포) → 3 컬러` 에서 `1 콘텐츠(심문·목차화·상수화) → 2 레이아웃 → 3 타이포 → 4 컬러` 로.
- 신규 계약: 목차 트리(stage1)가 전 계층의 단일 참조이자 섹션/컴포넌트 경계의 원천. 하류는 섹션을 발명·통합·삭제할 수 없다.
- 문서 형식: 각 계층을 `서사(왜 이 순서, 5줄 이하) → 결정 축 표 → 무효 조건` 의 고정 3부로 쓴다. 게이트는 기계검사 가능한 문장만.

## 악용 경로와 차단 규칙

| # | 신규 규칙 | 악용/붕괴 경로 | 차단 |
|---|---|---|---|
| 1 | 상류 개정 루프 (하류 브리프의 "상류 개정 제안" → 상류 stage 갱신 → 현 계층 재개) | 무한 왕복: L2 제안 → L1 갱신 → L2 재개 → 또 제안 | 계층당 개정 1회 캡. 같은 계층에서 2회째 개정 신호 = 스팟 모드의 3건 규칙과 동형으로 "축 재결정 신호" 승격, 해당 상류 계층 문답으로 복귀 |
| 2 | 심문 문답 (애매 구간을 닫힌 해석 선택지로 사용자에게 확인) | 캡 우회: 애매 구간 12개를 전부 개별 질문으로 승격해 인터뷰 피로 유발 | 개별 질문은 체감 상위 3구간까지. 4번째부터는 추천 해석 표 1개로 묶어 일괄 승인 1문항 |
| 3 | L3 표현 장치 정리 (타이포 계층의 삭제 권한) | 경계 악용: "표현 장치"를 넓게 해석해 카피·콘텐츠 실체를 L3에서 삭제 (상류 무효화 역류) | 표현 장치 = 트리에 콘텐츠 노드로 존재하지 않는 것만 (장식 넘버링, 중복 아이브로우, 순수 타이포 장치)의 폐쇄 정의. 트리 노드 삭제는 무조건 상류 개정 제안 |
| 4 | 트리 계약 (stage1 트리 = 하류 전제) | 트리 없는 하류 진행: 구 회차 stage 파일로 계층 2+ 디스패치 | 계층 2+ 디스패치 무효 조건에 "stage1 트리 절 부재" 추가. 단 구 회차의 스팟 모드는 기존 stage 그대로 상속 (트리 소급 작성 금지 - 스팟은 좁고 빠르게가 존재 이유) |
| 5 | 튜토리얼 3부 형식 | 서사가 게이트를 잡아먹음: 정성 설명이 무효 조건·측정 규율을 풀어 써서 재디스패치 판정 기준이 흐려짐 (정성 언어는 단계 간 증발한다) | 3부 구성을 구조로 강제: 서사는 5줄 이하, 무효 조건은 기계검사 가능한 문장만 (수치·파일 존재·절 존재). 서사는 게이트를 요약·복창할 수 없고 순서의 이유만 말한다 |
| 6 | 상수화 (콘텐츠 트리의 데이터화) | 강제력 과대 광고: "하류가 계약에서 이탈 불가능"을 전 스택에 약속 | 스택별 강제력을 stage1 에 명시 기록: 프레임워크(데이터 모듈) = 구조적 강제 / 정적 HTML = 계약 파일 + 회귀 grep 규율. 비프레임워크에서 상수화는 개념 단위(마크업 그루핑·네이밍)로 유지 |

## 유지 확인 (전환으로 건드리면 안 되는 것)

- fix 스코프 = audit 전체 payload (strong 텔 + 3초:예 전수) - 계층 수가 늘어도 계층별로 동일 적용.
- 측정 2등급 (리터럴 실측 / 판독), 여백 직접 조절 금지, 처방 사다리 (재설계 > 삭제 > 축소 > 교체).
- 간격 래더 선언은 stage2 소유, stage3 은 상속 전용 (두 stage 가 간격 값을 각자 주장하면 드리프트).
- 성공 기준 분기: L1 = 트리 승인 + 심문 반영 (콘텐츠 변경분 있으면 체감도 확인), L2~4 = 3초 체감.
- strict 모드 (별첨 A)·5행 레이어 표·blueprint-author·fidelity-verifier·택소노미 SSOT·스캐너: 무수정 (5행 표는 판정용 측정 축이라 인터뷰 계층 수와 독립).
- 실패 교훈 문장 (thegot·deslop-v2·diff-01 등 회차 실증 인용) 은 재배치만, 삭제 금지.
