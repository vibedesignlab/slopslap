# Slop Fix

대중이 느끼는 "AI 티"를 계층별 사용자 문답으로 확정하며 지우는 작업 스킬. 역할 분리가 핵심이다:

- **진단 = `slop-auditor` 에이전트** (read-only, 독립 컨텍스트). 계층 하나씩 진단 브리프(문제·근거·수정 옵션·문답 후보)를 낸다. 생성한 쪽이 자기 검증하지 않는다.
- **문답·확정·적용 = 이 스킬** (메인 컨텍스트). 브리프를 사용자에게 설명하고, 문답으로 확정된 값을 stage 파일로 상수화한 뒤 적용한다.

100% 자동 판정-수정이 아니다. **각 계층의 기준점을 사용자가 문답으로 제공한다.** 판정자가 자기 기준점을 자기 작성하던 것이 과거 회차 실패들의 공통 뿌리였고, 이 구조에서는 사람이 그 기준점이다.

## 왜 이 순서인가 (계층 모델 - 전 계층의 서사)

화면은 콘텐츠의 논리 구조 위에 표현 계층이 쌓인 것이다. 콘텐츠가 무엇을 어떤 묶음·순서로 말할지 확정하면(계층 1), 레이아웃이 그 묶음들에 공간을 배정하고(계층 2), 타이포그래피가 위계를 활자로 표현하고(계층 3), 컬러가 시맨틱을 칠한다(계층 4). AI slop 의 가장 깊은 서명은 이 순서의 역행 - 콘텐츠가 결정되기 전에 스타일이 먼저 발린 화면이다. 그래서 수정도 이 순서로만 안정적이며, 계층 1 의 목차 트리가 이후 전 계층이 공유하는 단일 참조가 된다.

**계층 고정 명칭 (구속 - 브리지·질문·보고서·stage 파일이 전부 이 명칭만 쓴다)**: ① **기획에 따른 콘텐츠 구조화** ② **레이아웃 최적화** ③ **타이포 위계 정리·불필요한 텍스트 제거** ④ **컬러 스킴 최적화**. 사용자가 지금 4단계 여정의 어디에 있는지를 모든 대면 출력이 이 명칭으로 알린다.

## 핵심 원칙

- **리덕티브 결정트리가 최우선 (신설 계약).** AI-slop 수정의 기본 동작은 덧칠(누적)이 아니라 걷어냄이다. 매 라운드 순서는 고정이다: (1) AI slop 이 만든 불필요 요소를 최소화한다 → (2) 가장 무난한, 평균 이상으로 안정된 baseline 으로 복귀시킨다 → (3) 킥(차별화 액센트)은 그 baseline 위에 나중에 딱 하나만 고려한다 - 지금 라운드에서 새 요소 추가는 금지. 처방 우선순위는 **삭제 > 통합 > 축소 > 교체**이고 신규 요소 추가는 사다리에 없다 (foundation 도출불가 결함만 예외적으로 재설계가 그 위). 여러 결함이 얽혔으면 개별로 하나씩 치지 말고 "가장 먼저 지울 루트 → 그 삭제가 연쇄로 해소하는 하위 결함"을 트리로 잡아 상위 삭제부터 집행한다 (빈 래퍼를 걷으면 그 안의 자잘한 스타일 텔이 통째로 사라지는 식). 작은 페이지에 계층당 순차 서브에이전트로 20분 화려하게 덧칠하며 누적시키는 것이 이 계약이 죽이는 실패다.
- **삭제는 구성의 시작이지 끝이 아니다 (positive composition, 신설 계약).** 리덕티브가 요소를 걷어내는 것이라면, 그 뒤 남는 공간의 재구성까지가 한 처방이다 - 삭제 → 재균형 → 의도적 여백·정렬·그루핑이 한 라운드의 완결 단위다. 요소(목업·필러 카드·좌border·중복 CTA)를 뺀 채 빈 구멍을 남기면(`unbalanced-void-after-deletion`) "깨끗하게 비었다" 가 아니라 "덜 만들다 만" 미완으로 읽힌다 - 시니어 UI 디자이너의 눈은 삭제된 깨끗함이 아니라 의도적으로 구성된 균형을 본다. 2열/히어로에서 한쪽을 제거하면 남은 열의 재중앙·재비율(2열 → 1열 폭 재산정)·재그루핑을 같은 처방에 명기하고, 한쪽 쏠림(`left-skewed-uncomposed`)·근거 없는 균등분할(`default-equal-thirds`)은 우선순위 기반 비대칭 구성으로 재비율한다. **단 재구성 != 장식 추가** - 오브·그라디언트·새 카드로 구멍을 메우는 것은 금지이고(신규 요소 추가는 위 리덕티브 사다리에 없다는 계약과 일관), 오직 기존 콘텐츠의 재배치·재정렬·재중앙·재비율만 허용한다.
- **계층 의존성 고정: 1 콘텐츠 → 2 레이아웃 → 3 타이포 → 4 컬러.** 상류 확정 없이 하류 진행 금지. 상류가 바뀌면 하류 브리프는 무효이므로 순서를 건너뛰지 않는다.
- **목차 트리가 전 계층의 단일 참조다 (신설 계약).** stage1 의 트리 노드가 곧 섹션·컴포넌트 경계다(비프레임워크 스택은 개념 단위로라도). 하류 계층은 섹션을 발명·통합·삭제할 수 없다 - 그 필요가 발견되면 "상류 개정 제안"으로 올린다(아래 상류 개정 루프). 트리에서 같은 형태의 형제 노드 반복 = 반복 컴포넌트의 단서이고, 반복 없는 노드에 반복형 UI 를 씌우면 콘텐츠-레이아웃 부정합이다.
- **확정 상수가 계약이다.** 문답으로 확정된 값은 즉시 회차 폴더의 `stage<N>-*.md` 에 기록한다. 이후의 적용·회귀·재점검은 이 파일만 읽는다 - 기준이 디스패처의 기억(프롬프트)에만 있으면 세션 경계에서 승계가 무너진다.
- **fix 스코프 = audit 전체 payload, 문답이 아니다 (최우선 계약).** 브리프의 **모든 strong 텔 + 모든 "3초: 예" 항목**은 그 계층의 수정 대상에 반드시 포함된다. 문답에 오르지 않았다는 이유로 strong 텔을 미적용하는 것은 이 스킬의 정의된 실패다 (diff-thin: "audit 10분 돌리고 곁가지 3개만 지움"). 문답은 이 항목들을 **뺄 수 없다** - 문답이 결정하는 것은 오직 (a) 여러 처방 옵션 중 어느 전략을 택할지, (b) 적용 순서·우선순위뿐이다.
- **질문은 큰 결정에서 전략을 정리하는 것 (층위 규율).** 문답은 상류 전략 결정부터 내려온다. weak·device 텔(넘버링 삭제, 비활성 폼 제거, 라벨 교체 류)은 **개별 질문으로 승격 금지** - "이 자잘한 것들은 정리하겠습니다" 한 줄 통보로 처리하고, 질문 지면은 전략 결정에만 쓴다. 장치 삭제 3개를 큰 질문인 양 3개로 부풀리는 것이 알려진 층위 실패다.
- **과감하게, 빼는 쪽으로.** 처방 사다리는 **재설계 > 삭제 > 축소 > 교체**. 재설계는 foundation 결함(콘텐츠에서 도출 불가한 색 체계·레이아웃 타입·섹션 리듬) 전용 최상단. 장치 결함은 삭제부터. 기믹 위젯 삭제·카피 축소·섹션 통합이 기본기고, 색만 바꾸는 소심한 재스타일링은 금지. **좌측 컬러 border 카드(`colored-left-border-cards`)는 삭제가 정답이고, 줄 색만 교체하는 재색은 티인 패턴을 그대로 남긴 회피이므로 금지** - 강조가 필요하면 좌border 가 아니라 figure-ground(명도·배경층 분리)·크기·여백 위계로 만든다. 위험은 계층별 문답이 막아준다.
- **성공 기준은 계층별로 분기한다.** 계층 2~4 는 "3초 안에 다른 페이지로 인식되는가"가 상위 기준 - 확정값 전 항목 달성 + 체감 변화 없음 = 이 스킬의 실패 (수치 뒤에 숨어 성공으로 팔지 말 것). **특히 계층 2 는 분할선 미러 통일(42/58·58/42 교대)을 그리드 수정 완료로 자평하는 것을 금지한다**(`undisciplined-grid`) - 세로 기준선이 섹션을 실제로 관통(관련 섹션 x 가 단일 목표값으로 수렴 + 앵커 존재)하는가만 성공이고, 미러 교대는 x 가 두 값으로 갈라지는 눈속임이라 자기인증이 차단된다. **계층 1 만 예외**: 성공 기준은 트리 승인 + 심문 반영(하류가 콘텐츠를 재해석하지 않고 디자인 결정을 완결할 수 있는 기반)이고, 심문으로 콘텐츠가 바뀌었으면 그 변경분의 체감 확인을 추가한다.
- **간격 래더 소유권: stage2.** 래더 선언(base + 티어)은 stage2 가 소유하고 stage3 은 상속 전용이다 - 두 stage 가 간격 값을 각자 주장하면 드리프트가 난다.
- **처방 출처는 SSOT, 런타임 로드.** 무엇이 슬롭이고 무엇으로 탈출하는지는 `src/data/aiSlopTaxonomyData.js` 의 tell/escape/escapeNote 에서 온다. 항목 수·버전·임계값을 이 문서에 하드코딩하지 않는다. 표현은 대상 프로젝트 자신의 토큰·관례로.
- **측정 2등급 준수.** 리터럴 실측(파일에 리터럴로 존재하는 값, file:line 의무)과 판독(렌더 계산 기하, `판독:` 표기)을 구분한다. 판독을 실측으로 파는 것은 위조.
- **위생은 스코프 밖.** px→rem, 토큰 정리, a11y 가드 같은 화면 안 바뀌는 수정은 명시 요청 시만. 단 **행간·자간·그루핑 간격·계층 마진은 위생이 아니라** 화면이 달라 보이는 계층 3 수정이다 (위생 강등 방치가 알려진 과거 실패). 마찬가지로 **radius 스케일 일관성(`unscaled-radius-scale`)과 의미없는 컨테이너 네스팅(`meaningless-container-nesting`)도 위생이 아니라** 화면·구조가 달라 보이는 계층 2 수정이다 - "px 값이라 슬롭 아님 / 0초 검사 제외" 는 오독이며(surf-tutorial 회차: radius 를 위생으로 빼서 검사조차 안 한 갭), radius 도 타이포·간격과 동급의 스케일 규율 대상이고 유령 래퍼는 삭제 대상이다.
- **CSS 모듈화 리팩토링 트랙 (신설 계약, "화면이 달라 보여야" 의 유일한 예외).** 인라인/임베디드 `<style>` 다수 + 외부 CSS 모듈 부재 + :root 토큰 미집약(`inline-css-no-modularization`)은 유지보수·디자인 시스템 부재 신호다. 처방은 **인라인/임베디드 CSS 를 외부 모듈로 분리(권장 4분할: reset / tokens / layout / components)하고 :root 디자인 토큰(색·간격·타이포·radius)을 tokens 모듈에 집약**해 단일 진원지를 만드는 것. 이 트랙만은 다른 계층과 달리 **렌더가 동일해도 성공** - 산출물이 "화면이 달라 보여야" 하는 계약(계층 2~4)의 명시적 예외이며, 성공 기준은 모듈 분리·토큰 집약 완료 + **렌더 패리티(before/after 시각 동일)** 다. 위치: 계층 2(레이아웃)의 **준비 단계**로 먼저 돌리거나(이후 계층 2~4 의 시각 수정이 이 토큰 위에서 이뤄지는 기반), 시각 수정과 섞기 부담스러우면 **별도 리팩토링 스텝**으로 분리한다. 이 트랙의 수정은 계층 2 의 3초 체감 성공 판정·payload 전수(strong 텔·3초 예) 집계에 넣지 않는다(렌더가 안 바뀌므로 3초로 잴 수 없다). 프레임워크 스코프드 스타일(CSS Modules·styled-components·Vue scoped·Tailwind)·의도된 self-contained 산출물(Artifact 류 CSP 인라인)은 모듈화의 다른 형태이므로 이 트랙 대상이 아니다.
- em-dash(U+2014) 0건 (SSOT 원문 인용부 예외).

## 절차

### 0. 준비
- 대상 스택·토큰 체계 파악, 회차 폴더 생성 (`<대상 인접>/<대상명>-audit/round-YYYY-MM-DD_HHMM/`).
- 버전 보존: 대상이 버전관리 밖이거나 원본 보존을 원하면 `<대상>-deslop` 복사 → git init → baseline 커밋. 이미 git 이면 브랜치. **수정 1건 = 커밋 1개.**

### 0.5 의도 인터뷰 (회차당 1회, 전 계층의 최상류 - 생략 금지)
계층 문답보다 위에 있는 **회차 전체의 의도**를 먼저 계약한다. 닫힌 질문 3개 + 추천 기본값, 한 번에:
1. **컨셉 선언**: 무드 + 대상 + 경험 동사 1문장을 사용자가 확정한다 (도구가 콘텐츠에서 역산한 초안을 추천으로 제시). 이후 모든 계층의 "도출되는가" 판단 기준이 역산이 아니라 이 확정 문장이 된다.
2. **과감도**: 전면 재설계 허용(토대 재결정 포함) / 구조 조정까지(섹션 통합·삭제·재배치) / 표면 정리만. **이 답이 스킬의 권한 계약이다** - "재설계 허용"이면 사다리 최상단 처방을 계층마다 되묻지 않고 적용하고, "표면 정리만"이면 구조 처방은 제안만 하고 적용하지 않는다.
3. **불가침 요소**: 건드리면 안 되는 시그니처(로고 연출·특정 카피·브랜드 장치)를 사용자가 지정. 이후 모든 처방에서 예외로 보호하고, audit 이 불가침 요소를 텔로 지목하면 "waive(사용자 지정)" 로 기록한다.

확정 즉시 `stage0-intent.md` 로 저장하고, 이후 모든 계층 디스패치 프롬프트에 이 파일 경로를 상류로 넘긴다. (stage0 은 회차의 의도, 계층 1 의 심문은 콘텐츠 각 구간의 의미 - 서로 다른 층이며 stage0 이 심문을 대체하지 않는다.)

**로드맵 제시 (stage0 저장 직후 1회)**: 사용자에게 4단계 지도를 보여준다 - "① 기획에 따른 콘텐츠 구조화 → ② 레이아웃 최적화 → ③ 타이포 위계 정리·불필요한 텍스트 제거 → ④ 컬러 스킴 최적화. 각 단계는 진단 보고 → 전략 문답 → 적용 → 완료 보고로 진행됩니다." 이후 모든 계층 브리지가 이 지도 위의 현 위치를 가리킨다.

### 1~4. 계층 루프 (계층마다 동일한 9박자)

⓪ **브리지 (튜토리얼의 실체)**: 계층 시작 전에 사용자에게 현 위치 + 3줄 요약을 제시한다 - "**▶ <n>/4 <고정 명칭>** (완료: <이전 단계들> / 남음: <이후 단계들>). 지금까지 확정: <상류 stage 핵심값>. 이번 계층이 결정할 것: <결정 축 목록>. 왜 지금인가: <계층 모델에서 한 문장>." 계층 1 은 상류가 stage0 뿐이므로 의도 인터뷰 요약이 브리지다.
① **디스패치**: slop-auditor 에게 계층 번호로 브리프를 받는다. 프롬프트에 반드시 - 대상 경로·스택·토큰 위치, 계층 번호, 상류 stage 파일 경로(**stage0-intent.md 는 전 계층 필수**, 계층 2 는 +stage1, 계층 3 은 +stage1+2, 계층 4 는 +stage1+2+3), 회차 폴더(브리프 출력처), SSOT 경로. **계층 2+ 디스패치 전에 stage1 의 목차 트리 절 존재를 스킬이 확인한다** - 트리 없이 하류 디스패치 금지. **무효 조건**(브리프 파일 부재 / 결정 축 절 부재·빈 축 / 문답 후보가 축 밖에서 나옴 / 수정 옵션에 3초 표기 누락 / 실측·판독 구분 없는 수치 / 계층 1: 심문 후보 절 부재·해석 선택지 없는 심문 후보 / 계층 2+: 섹션 통합·삭제를 상류 개정 제안이 아닌 자기 계층 처방으로 냄)이면 재디스패치. **계층별 순차 디스패치가 원칙** - 4계층을 미리 한꺼번에 뽑으면 하류가 상류 확정값 없이 역산으로 돌아간다 (brief-all 은 수정 없는 "점검만" 요청 전용). 참고: 과감도가 재설계 허용/구조 조정인 회차의 브리프는 위반 증거가 표본 2~3개로 프루닝되어 온다 (auditor 의 수치 2종 분리 규율) - 표본이 적다고 재디스패치하지 말 것, 전수 대조는 ⑦ 회귀의 몫이다.
② **계층 진단 보고서 (점검 결과를 딱 보여주는 시점 - 문답의 게이트, 이 보고서 제시 전 어떤 질문도 금지)**: `brief-stage<N>.md` 를 직접 읽고(요약 전달 금지 - SSOT id·file:line 유실 경로) 고정 규격으로 사용자에게 보고한다:
  1. 로드맵 현 위치 (▶ n/4 고정 명칭)
  2. 총평 1문장
  3. 결정 축 표 (현재 판독 → 추천 → 근거)
  4. 문제 Top + 증거 (**SSOT id·file:line 보존 의무** - 재서술은 허용, 증거 탈락은 열화 요약으로 무효)
  5. **전략 프레임** (필수 3요소: 이 계층에서 화면 문제의 구조 + 추천 전략 + 그 전략을 구성하는 결정 목록. 한 줄 걸치기 = 요식화 무효)
③ **문답 (전략 세션 - 미시 질문 몇 개 던지고 끝나는 것이 아니다)**: 3단 고정 진행.
  - **1단 전략 승인**: 보고서의 전략 프레임에 대한 방향 확인 1문항 ("이 전략으로 갈까요, 아니면 <대안 방향>?"). 계층 1 의 1단은 stage0 컨셉 선언을 인용해 연다.
  - **2단 축 캐스케이드**: 결정 축들을 승인된 전략의 구성 결정으로서 의존 순서대로 - **앞 답이 뒤 질문의 전제로 인용된다**. 계층 1 은 심문 후보가 1단과 2단 사이에 온다 (전략 승인 → 심문 → 축).
  - **3단 파생 좁히기 (2스텝 퍼널, 최대 2문항)**: 축 답이 여는 하위 결정을 좁힌다 - 예: 3초 핵심 = "4-in-1 차별점" 확정 → "그럼 기존 상호명은 어디로 강등할까 (로고 존만 / 서브라인)". 파생도 결정 수준(강등 위치·적용 범위·예외 지정)만.
  **질문 4요소 고정 서식 (전 질문 의무)**: `[계층 n/4 · <고정 명칭> | 축: <축명>]` 위치 태그 + **상류 확정 인용** ("계층 1에서 읽기 모드 = 탐색으로 확정하셨습니다") + 닫힌 선택지 + 추천 기본값 + **"이 답이 결정하는 것"** 1줄. **상류 확정(stage 값·전략 프레임·앞 축 답)을 인용하지 못하는 질문은 가이드 밖에서 발명된 질문이므로 그 자체가 무효다.** 실행 수준 프레이밍("H1 을 뭘로 바꿀까요") 금지 - 결정 수준으로. **화면이 깨끗한 계층에서도 축은 묻는다** (프레임 확정이 목적이지 문제 해결이 아니다). 문제 목록에서 질문 생성 금지. 브리프의 "정리 예정" 목록은 질문 없이 "다음은 일괄 정리하겠습니다" 한 줄 통보. **strong 텔·3초:예 항목의 적용 여부 자체를 묻는 질문도 금지** - 그건 이미 fix 대상이고, 문답이 정하는 것은 그 적용의 전략뿐이다. 질문 캡: 전략 승인 1 + 축(3~4) + 파생(≤2), 라운드 최대 2회 - 그 이상은 인터뷰 피로이고, 남는 모호함은 과감도 계약 안에서 스킬이 결정하고 결과로 보여준다.
④ **확정 기록**: 확정 즉시 stage 파일 작성 (아래 규격). 이때 그 계층 브리프의 strong 텔·3초:예 항목을 **전수 나열**하고 각 항목에 (택한 전략 / 적용 예정) 을 붙인다 - 이 목록이 ⑤ 적용의 체크리스트이자 ⑦ 회귀의 대조표다.
⑤ **적용 (payload 전수)**: ④ 체크리스트의 **전 항목**을 확정 전략대로 적용한다. 질문에 오른 항목만 고르는 것은 스코프 계약 위반. 상류부터, 커밋 단위, 최소 diff 가 아니라 과감한 diff - 삭제는 관련 CSS·JS·orphan-ref(색-명명 클래스, 미사용 var·키프레임)까지 함께. 확정 전략 밖의 신규 결정은 만들지 않되(스코프 크리프 금지), 확정된 payload 는 하나도 빠뜨리지 않는다. **불가능한 목표는 이행하지 말고 정지**: 확정값이 물리 불변식(이미지 렌더 폭 ≤ 원본 natural 폭 등)과 충돌하면 그 항목을 정지하고 충돌을 보고한다.
⑥ **성공 확인 (계층별 분기)**: 계층 2~4 - 렌더 가능하면 before/after 스크린샷 1쌍, 불가하면 diff 요약으로 "3초 안에 다른 페이지인가"를 사용자에게 확인받고 기록. 계층 1 - 목차 트리와 심문 반영을 사용자에게 승인받고, 심문으로 바뀐 콘텐츠가 있으면 그 변경분만 체감 확인.
⑦ **회귀 (리터럴만)**: stage 파일의 실측 가능 값을 전수 grep 대조 (간격 래더는 전 padding/margin/gap 리터럴, 래더 밖 = 0 이 게이트; 이미지는 렌더 폭 vs naturalWidth, 업스케일 = 0). **진단이 표본으로 프루닝된 회차도 회귀는 전수다** - 전수 실측의 유일한 집이 여기다. 코드 텔은 `node scripts/scan-slop-signals.mjs <경로>` 재실행. 계층 2+ 는 화면과 트리의 대응도 대조한다 - 트리에 없는 섹션이 화면에 있으면 위반. 미달 항목은 "미완료" 표기, 통과 위장 금지.
⑧ **계층 완료 보고서 (다음 계층 브리지 전 제시)**: 고정 규격 - 확정값 요약 (stage 파일 핵심) / payload 체크리스트 전수 대조 결과 (④ 목록의 항목별 적용 여부) / 회귀 결과 (**미달 = "미완료" 그대로 노출 - 보고서 서사가 회귀 결과를 가리면 보고서 무효**) / before-after (계층 1 은 트리·심문 반영 승인으로 대체).

### 상류 개정 루프 (하류가 상류 수정 필요를 발견했을 때 - 유일한 역류 경로)

하류 브리프의 "상류 개정 제안" 절(예: 계층 2 가 섹션 통합을 제안) 또는 적용 중의 발견은 이 절차로만 처리한다:
1. 제안을 사용자에게 그대로 올려 승인받는다 (해당 상류 축의 재확정 문답 1건).
2. 승인되면 상류 stage 파일을 갱신하고, 현재 계층을 **경량 재디스패치**한다 (개정분 반영만 - 전체 재진단 금지).
3. **계층당 개정 1회 캡.** 같은 계층에서 2회째 개정 신호가 나오면 그것은 스팟이 아니라 축 재결정 신호다 - 해당 상류 계층의 문답으로 정식 복귀한다.

### 계층별 규격 (각 계층: 서사 → 결정 축 → 기록 → 구속)

**계층 1 - 콘텐츠 (의도 심문·목차화·상수화) → `stage1-contents.md`**
- 서사: 하류 전부가 이 계층의 산출물을 입는다. 애매한 카피는 그 자체가 카피 슬롭의 정체이므로, 의미를 심문으로 확정하고 그 의미를 목차 트리로 구조화한 뒤 상수화한다. 트리가 섹션·컴포넌트 경계의 원천이다.
- 결정 축: **심문 후보** (축 이전, 질문의 1원천 - 실물 카피·이미지의 애매 구간마다 "하고 싶은 얘기가 A 인가 B 인가" 해석 선택지 + 추천. **개별 질문은 체감 상위 3구간까지, 4번째부터는 추천 해석 표 1개로 일괄 승인 1문항**) / ① 3초 핵심 ② 읽기 모드 (핵심 오퍼 수 N 과 전환 목표에서 도출: N≤5+설득+전환=몰입 스토리텔링, N≥8 비교 항목=탐색, 장문 본문=선형 읽기 - 현 화면 생김새를 근거로 쓰는 순환 금지) ③ 섹션 우선순위·순서 ④ **목차 트리 승인** (섹션 경계·묶음·반복 노드).
- 기록: 심문 기록 (구간 → 질문 → 확정 의미, 일괄 승인분 포함) · 컨셉 선언 연결 · N·전환 목표·읽기 모드(도출 근거) · **목차 트리** (노드당 `id / 유형(섹션|컴포넌트|요소) / 반복 여부 / 콘텐츠 참조(확정 카피·이미지 id)` 의 중첩 개요) · 확정 카피 전문 (삭제 승인분은 "삭제 태깅" 별도 목록 - 계층 3 이 집행할 수 있다) · 이미지 인벤토리 (id·파일·내용 1줄·정보밀도 고/중/저·natural WxH px 실측 의무) · **상수화 선언**: 스택별 실체 (프레임워크 = 데이터 모듈 경로에 트리를 코드로 추출, 정적 HTML = 이 stage 파일이 계약이고 마크업 그루핑·네이밍이 개념 단위) + 강제력 수준 명시 (구조적 강제 / 회귀 grep 규율).
- 구속: 트리 절 없는 stage1 은 무효 (하류 디스패치 불가). 심문은 열린 질문("무슨 뜻이에요?") 금지 - 반드시 해석 선택지 제시.

**계층 2 - 레이아웃 안정화 → `stage2-layout.md`**
- 서사: 트리가 준 섹션 단위에 공간 전략을 입힌다. 섹션의 존재·경계는 이미 상류가 확정했으므로 이 계층은 발명하지 않고 배정한다.
- 결정 축: ① 레이아웃 타입 선언 (stage1 읽기 모드에서 도출된 추천 + 근거를 제시하고 승인받는다 - bare "fluid 로 변경" 금지) ② 분할 전략·앵커 (분할선 수렴 기준 섹션 - **관통선 판정**(`undisciplined-grid`): 관련 섹션의 세로 분할선 x 가 단일 목표값으로 수렴하고 앵커 섹션이 존재하는가. 42/58·58/42 미러 교대는 x 가 두 값으로 갈라지는 눈속임이라 그리드가 아니다) ③ 정보 밀도·여백 톤 + **간격 래더 선언** (base + 티어 내부<그룹<블록<섹션, 각 ≥1.5× - 이 계층 소유, stage3 은 상속) ④ **radius 스케일 선언**(`unscaled-radius-scale`): 화면에 쓰인 border-radius 고유값을 전수 나열해 종수를 세고, 2~3단 토큰 스케일(예: sm/md/lg + pill)로 선언해 전 요소를 거기서 도출한다 - 개별 반경 눈대중 금지, radius 도 타이포·간격과 동급의 스케일 규율 대상이다 ⑤ **DOM 네스팅 정리**(`meaningless-container-nesting`): 레이아웃 기여 0 인 빈 래퍼(단일 자식만 감싸고 grid/flex·여백·배경·시맨틱 역할 어느 것도 부여 않는 유령 div, CSS 셀렉터 없는 className)는 삭제·언랩. 트랙·여백·시맨틱 역할 중 하나라도 실제 부여하는 래퍼만 유지. ⑥ **양의 구성 (positive composition, 삭제 후 재구성)**(`unbalanced-void-after-deletion`·`left-skewed-uncomposed`·`default-equal-thirds`): 이 계층은 삭제만 결정하는 것이 아니라 삭제 후 남는 공간의 재구성까지 결정한다. 삭제 처방(목업·필러 카드·좌border 제거 등)에는 반드시 "삭제 후 남는 공간의 재구성 목표"를 동반한다 - 남은 열의 재중앙·재비율(2열 → 1열 폭 재산정)·재그루핑을 같은 항목에 명기하고 void 를 방치하지 않는다. 무게중심이 한쪽에 쏠려 반대쪽이 의도 없이 빈 섹션, 근거 없이 균등 N등분한 영역은 우선순위 기반 비대칭 구성으로 재비율한다 (재구성 != 장식 추가, 기존 콘텐츠 재배치만). ⑦ **CSS 모듈화 (design refactoring, 위생 예외 트랙)**(`inline-css-no-modularization`): 인라인/임베디드 CSS 를 외부 모듈로 분리(reset/tokens/layout/components 4분할)하고 :root 토큰을 tokens 모듈에 집약한다. 이 축은 렌더가 동일해도 성공(위 CSS 모듈화 리팩토링 트랙 계약) - 계층 2 준비 단계 또는 별도 스텝으로 집행하고, 3초 체감·payload 전수 집계에서 제외한다. 프레임워크 스코프드 스타일은 대상 아님.
- 기록: 타입 선언문(+근거 1문장) · 전 캔버스 분할 스킴 (트랙 정의, 비게 될 공간의 소속 트랙까지) · **관련 섹션 분할선 x 목표값·앵커 섹션 명시** · 블록 배치 규칙 (트리 노드별 col start/end·얼라인 라인 - **대상 스택의 그리드 프리미티브로 번역해 제시**: MUI Grid / CSS grid / 개념상 트랙) · 간격 래더 선언 · **radius 스케일 선언** (토큰 티어 + 요소별 매핑) · **네스팅 정리 목록** (삭제·언랩할 유령 래퍼를 전수 나열) · **삭제 후 재구성 목표** (삭제 항목별로 남는 공간의 재중앙·재비율·재그루핑 목표를 명기 - void 방치 처방 금지) · 섹션별 밀도 목표 (뷰포트당 아이디어 수, 수치 - "저밀도" 류 정성어 무효) · **CSS 모듈화 목록** (design refactoring 축이 발화하면: 분리할 인라인/임베디드 스타일 -> 목표 모듈 4분할 매핑 + tokens 로 집약할 :root 토큰 목록 + 렌더 패리티 확인 방법. 이 목록은 3초 체감 payload 와 분리해 별도 트랙으로 표기).
- 구속: **여백 직접 조절 금지.** "패딩을 Npx 로" 식 목표는 기록 자체가 무효다 - 분할 스킴과 배치 규칙을 고치면 여백은 잔여로 따라온다. 레이아웃 포화: 캔버스 전체가 트랙에 소속 (일부만 그리드, 나머지 방치 = 커버리지 실패). **미러 = 성공 아님 (자기인증 차단).** 분할선 미러 통일(42/58 교대)만으로 계층 2 성공 처리 금지 - 세로 기준선이 섹션을 실제로 관통(단일 x 수렴 + 앵커)하는가로만 판정한다. **radius·네스팅 처방은 리덕티브**: 유령 래퍼는 삭제·언랩이 정답이지 스타일을 새로 붙이는 것이 아니고, 개별 반경 눈대중이 아니라 스케일 토큰에서만 도출한다. **"깨끗하게 비었다" 를 성공으로 팔지 말 것 (양의 구성 구속).** 요소를 삭제한 뒤 남은 void 를 재구성하지 않으면 계층 2 미완이다 - 삭제 → 재균형 → 의도적 여백·정렬·그루핑까지가 한 라운드이고, 재구성은 장식 추가가 아니라(오브·그라디언트·새 카드 금지) 기존 콘텐츠의 재중앙·재비율·재그루핑만이다. **섹션 통합·삭제 필요 발견 = 상류 개정 제안** (자기 처방 금지 - AI slop 은 어지간하면 정보밀도를 높이는 쪽으로 실수하므로 이 제안은 흔하고 정상이다).

**계층 3 - 타이포그래피 → `stage3-typography.md`**
- 서사: 트리의 위계를 활자로 표현한다. 제목 레벨은 트리 깊이의 함수이고, 스케일 선언 하나에서 전 역할이 도출된다 - 타이포만 정리되어도 대중 체감의 절반이 잡힌다.
- 결정 축: ① **타이포 스케일 선언 승인** (base×r + 역할별 도출 - 선언 하나로 전 역할이 도출되므로 **문답 1문항으로 끝난다**. 대역·r 허용치는 SSOT note 가 원본) ② 시맨틱 레벨 도출 (트리 깊이 → h1~h6·role 매핑 승인) ③ 표현 장치 정리 방침 (아래 폐쇄 정의 안에서).
- 기록: 스케일 선언 (base×r, **stage2 래더 상속 명시**) · 트리 깊이→시맨틱 레벨 도출표 · 역할별 도출표 (size / line-height / letter-spacing / margin 위·아래 - 계층 근접: 제목 위 여백 ≥ 2× 아래 여백) · 표현 장치 정리 목록 · stage1 삭제 태깅분 집행 목록.
- 구속: **표현 장치의 폐쇄 정의** - 트리에 콘텐츠 노드로 존재하지 않는 것만 (장식 넘버링, 중복 아이브로우, 순수 타이포 장치). **트리 노드(콘텐츠 실체)의 삭제는 무조건 상류 개정 제안** - 이 계층의 삭제 권한은 stage1 삭제 태깅분의 집행 + 표현 장치까지다. 선언 없는 역할별 값 나열은 무효 (개별 값 눈대중의 재생산).

**계층 4 - 컬러 전략 → `stage4-color.md`**
- 서사: 확정된 구조와 위계 위에 마지막으로 시맨틱을 칠한다. 색이 마지막인 이유: 색은 역할의 함수인데 역할은 상류 3계층이 정한다.
- 결정 축: ① 콘텐츠가 실제 요구하는 기능 역할 수(브랜드 + CTA + 실재하는 시맨틱 상태만) vs 현재 hue 수를 나란히 제시 ② 시맨틱 팔레트 표(역할→색) 확정 ③ 형제·정성 콘텐츠에 매핑된 컬러의 해제 승인 - **형제 열거 = 트리의 형제 노드** (카테고리·특성 나열은 상태가 아니다). 채도·명도는 면죄부가 아니다, 검사는 도출 가능성이다.

- 구속: 적용 시 orphan-ref 스윕 필수. 모션·표면 장식(글로우·글래스·장식 도트)의 삭제 승인도 이 계층 말미에.

### 6. 2차 스팟 수정 (1차 완료 후, 사용자가 지점을 직접 짚을 때)
1차 회차의 확정 계약 위에서 도는 **좁고 빠른 후속 수정**이다. 1차 대비 분석 범위·시간을 대폭 줄이는 것이 이 모드의 존재 이유다:
- **전부 상속, 재확정 없음**: stage0(의도·과감도·불가침)~stage4 를 그대로 계약으로 쓴다 (구 회차의 stage 구성이 3계층이면 그 파일들 그대로 - 트리 소급 작성 금지). 의도 인터뷰·결정 축 질문을 다시 하지 않는다. 질문은 원칙 0개 - 유일한 예외는 요청이 **기존 확정 축과 충돌**할 때(예: "히어로를 더 크게" 가 스케일 캡과 충돌) 그 충돌 1건만 묻는다("기존 계약 X 를 깨고 진행할까요?").
- **진단은 스팟만**: 요청된 요소/섹션의 코드 구간 + 관련 stage 값만 읽는다. 전 화면 재점검·전 계층 루프 금지. 요청이 명확하면("이 버튼 색") 에이전트 디스패치 없이 메인 컨텍스트에서 직접 진단·적용한다. 요청이 정성적이면("히어로가 아직 어수선해") slop-auditor 를 **스팟 디스패치**한다 - 프롬프트에 "스팟: <요소>" + stage 파일 전부 + 회차 폴더. 산출물은 `brief-spot-<n>.md` (문제 최대 3, 결정 축 표 없음).
- **적용·검증도 스팟만**: 수정은 기존 계약값(스케일·래더·앵커·팔레트·트리)에서 도출한 값으로만. 회귀는 그 스팟 관련 값만 재확인, 렌더 대조도 해당 구간 크롭 1쌍. CHANGELOG 에 "2차 스팟" 절로 append.
- 스팟 수정이 3건 이상 쌓이거나 스팟들이 같은 축을 반복 건드리면, 그것은 스팟이 아니라 축 재결정 신호다 - 해당 계층 문답으로 승격을 제안한다.

```
## 슬롭 수정: <스코프> <날짜>
| 계층 | 확정 파일 | 적용 | 성공 확인 (1: 트리 승인 / 2~4: 3초) |
|---|---|---|---|
회귀: <grep/스캐너 결과 또는 스크린샷 경로>
상류 개정: <발생 계층·승인 여부·갱신 파일 | 없음>
보류: <사용자 거부 / 미확정 / 타 계층 예고 잔여>
```

## 흔한 함정

- **위생으로 분량 채우지 말 것.** 화면이 안 바뀌는 수정을 성과로 팔면 이 스킬의 존재 이유가 죽는다.
- **하류부터 고치지 말 것.** 콘텐츠 확정 전의 레이아웃 문답, 레이아웃 확정 전의 재도색은 작업 증발이다.
- **트리 없이 하류로 가지 말 것.** stage1 에 목차 트리 절이 없으면 계층 2+ 디스패치는 무효다 - 하류가 콘텐츠를 재해석하기 시작하는 순간 단일 참조 계약이 죽는다.
- **상류 개정을 왕복시키지 말 것.** 계층당 개정 1회 캡. 2회째 신호는 축 재결정으로 정식 승격 - 경량 루프를 문답 우회로로 쓰는 것이 이 규칙이 죽이는 악용이다.
- **심문을 에세이로 만들지 말 것.** "이 문장 무슨 뜻이에요?" 류 열린 심문 금지 - 반드시 해석 A|B 선택지 + 추천을 제시한다. 축 문답도 동일 (닫힌 질문 + 추천 기본값). 사용자가 디자이너가 아니라는 전제가 이 스킬의 출발점이다.
- **보고서 없이 질문으로 직행하지 말 것.** 계층 진단 보고서가 문답의 게이트다 - 사용자는 진단 전모를 본 뒤에만 질문을 받는다. 브리프 도착 즉시 낱개 질문을 던지는 것이 "미시 질문 몇 개 던지고 끝"의 구조적 원인이었다.
- **전략 프레임을 요식으로 걸치지 말 것.** 한 줄 전략 선언 뒤 축 질문 직행은 전략 세션이 아니다 - 프레임은 문제 구조 + 추천 전략 + 구성 결정 목록 3요소를 갖추고, 1단에서 방향 승인을 실제로 받는다.
- **표본을 통과 증거로 쓰지 말 것.** 프루닝된 표본은 위반 예시 전용이다. 어떤 축을 "통과"로 판정하려면 전수가 필요하다 (스팟 수정에서 메인 컨텍스트가 직접 진단할 때도 동일).
- **소심한 교체로 도망가지 말 것.** 삭제가 가능한 자리에서 교체를 고르면 안 된다. 사다리는 위에서부터.
- **확정 없이 진행하지 말 것.** 사용자 무응답 상태에서 "추천값으로 가정하고 적용"은 금지 - 추천값 일괄 승인 여부 자체를 물어라.
- **여백을 직접 조절하지 말 것.** 여백값 눈대중 조절은 조절된 여백(unpartitioned-space)을 재생산하는 짓이다.
- **수치 뒤에 숨지 말 것.** stage 전 항목 달성이 곧 성공이 아니다 - 체감이 상위 기준. 알려진 실패 사례: 2,429줄 페이지에 diff 55줄 내고 자축 - 사용자 판정은 "수정이 너무 미비했다".
- **foundation 결함을 장치 삭제로 때우지 말 것.** 색 체계·레이아웃 타입이 콘텐츠에서 도출 안 되는 화면에서 오브·글래스만 지우면 "배경만 바뀐" 결과가 된다 (thegot 회차 실증).
- **미러 통일을 그리드 완성으로 팔지 말 것.** 42/58·58/42 를 섹션마다 뒤집는 미러 교대는 세로 기준선이 섹션을 관통하지 않는 눈속임이다 - 분할선 x 를 단일 목표값으로 수렴시키고 앵커를 지정해야 그리드다 (surf-tutorial 회차: 미러를 "그리드 수정 완료" 로 자평한 실패).
- **덧칠로 고치지 말 것 (리덕티브 위반).** 매 라운드 화려한 요소를 누적하는 것은 slop 을 더 만드는 짓이다 - 가장 먼저 지울 루트를 트리로 잡아 삭제 > 통합 > 축소 > 교체 순으로 걷어내고 평균 이상 안정 baseline 으로 복귀시킨다. 킥은 그 위에 나중에 딱 하나, 지금 라운드의 신규 요소 추가는 금지.
- **삭제 후 void 를 방치하지 말 것 (양의 구성 위반).** 요소를 뺀 자리에 빈 구멍(히어로 우측 void 류)을 남기고 "깨끗하게 비었다" 로 완료 처리하는 것은 미완이다 - 삭제는 구성의 시작이지 끝이 아니며, 남은 콘텐츠를 재중앙·재비율·재그루핑해 균형을 회복해야 한 라운드가 끝난다 (surf-tutorial 사이클1 실증: 삭제만 하고 재구성을 안 해 시니어가 미완으로 판정). 단 재구성은 오브·그라디언트·새 카드가 아니라 기존 콘텐츠 재배치만.
- **대상 관례를 홈 레포 기준으로 덮어쓰지 말 것.** 다른 프로젝트의 idiom(순수 CSS hex 등)은 결함이 아니다.
- **경계**: 장문 한글 윤문 humanize-korean, 성능 frontend-perf-auditor, 레이아웃 안정성 심층 stable-layout-auditor, 타이포 상세 typography-auditor.

## 별첨 A: Strict 모드 (무인 실행·파이프라인 검증 전용, 명시 요청 시만)

사용자 문답 없이 돌아야 하는 회차(무인 일괄 처리, 파이프라인 자체의 회귀 검증)만 이 모드를 쓴다. slop-auditor 를 **strict** 로 디스패치해 전체 리포트(판정 티어 + 5행 레이어 표 + 처방표 + audit-report.md/html)를 받고, 아래 계약을 지킨다. 문답형 기본 경로에서 사용자가 하던 기준점 제공을 여기서는 블라인드 블루프린트가 대신한다.

1. **블라인드 블루프린트**: 콘텐츠 인벤토리 추출(텍스트 전문, 이미지 내용 서술 + 밀도 태그 + **natural WxH px 치수 의무**, 브랜드 자산 서술, 페이지 메타. CSS·클래스명·배치·스타일 어휘 금지 - 브랜드 색은 로고 자산 서술로만) → `blueprint-author` 디스패치(인벤토리·SSOT·baseline.md 출력 경로만; 대상 사이트 경로 포함 = 그 블루프린트 무효) → baseline 게이트: 모든 결정에 인벤토리 인용이 붙었고 **그 인용문이 인벤토리에 실존하는지 grep 대조**, 스토리보드 목표가 수치인지 확인, 미달 시 재디스패치 → 점검 디스패치에 baseline.md 경로를 넘긴다(diff 모드). baseline 은 targets.md 의 상위 문서다.
2. **점검 무효 조건 (재디스패치)**: 0단계 레이어 표 없이 돌아온 판정 / 시그니처 카운트 부재 / Top 에 3초 표기가 없거나 '예' 0개인데 정직 출구("가성비 수정 없음") 선언도 없음 / 리포트 파일 2종 부재 또는 레이어 표 5행 미완결.
3. **승인 게이트**: Top 수정안을 표(| # | 텔 | 재설계/삭제/축소/교체 | 영향 파일 | 기대효과 | 3초 | 위험도 |)로 승인받는다. foundation 재설계가 장치 삭제보다 위. '예' 0개인 표는 올리지 말고 더 과감한 재점검 먼저.
4. **targets.md**: 리포트의 처방표들(재설계·레이아웃·타이포)을 1:1 편입(재번역·요약 금지), 래더·토큰 집합 무손실 상속(값 탈락 = 합법 값이 유령 위반으로 둔갑, diff-01 실증), 이미지 목표는 트랙 %W + 렌더 px(≤ natural, 치수 명기) 2열 분리 - 업스케일 요구 목표는 편입하지 않고 충돌 보고(deslop-v2 실증). 스케일 선언문(base·r·래더 베이스) 없는 targets 는 무효.
5. **적용·회귀**: 상류부터 커밋 단위, orphan-ref 스윕, 전 간격 리터럴 래더 grep 전수(밖 = 0 게이트), 이미지 업스케일 = 0 게이트, 분할선 x 재계산·행간·그룹 간격 재측정. waive 항목은 "audit waive(사유) 유지" 로 인용 (범위 내로 위장 금지). 재점검 디스패치에는 targets.md 경로를 넘겨 그 수치로 재측정하게 한다.
6. **충실도 검증**: 구조 변경(그리드·히어로 재편) 포함 회차는 `prescription-fidelity-verifier`(독립 컨텍스트)에 audit-report.md·targets.md·원본·작업본 경로를 넘겨 처방↔diff 충실도를 검증한다. 수정 실행자가 자기 검증을 겸하지 않는다.

**반증 조건 (하나라도 터지면 변명 없이 구조 재검토, 2026-07-09 못박음)**: baseline 에 콘텐츠 인용 없는 결정이 무효 처리 없이 통용 / foundation 결함 확인 화면에서 baseline 이 현 화면과 사실상 동일(블라인드 실패) / diff 회차 Top 이 또 크롬-only.
