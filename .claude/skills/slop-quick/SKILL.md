# Slop Quick

문답 없는 단일 패스, **직설·기계적** 슬롭 제거. 이 스킬의 설계 철학:

> **그럴듯한 논리구조는 실행 AI가 오히려 오해한다.** 정성적 판단·다단계 결정 트리를 주면 실행 단계에서 증발하거나 왜곡된다. 그래서 이 스킬은 "AI 산출물이 **통계적으로** 이렇게 망친다 → **기계적으로** 이렇게 고친다" 는 직설 명령 5개만 든다. 판단이 아니라 규칙이다.

slop-fix 와의 경계: slop-fix 는 의도 인터뷰 + 4계층 문답 + 계층 상수화(느리고 깊다, 재설계용). slop-quick 은 아래 5규칙을 **유저가 풀기 전까지 기본으로 기계 적용**(빠르고 과감하다). 애매하거나 유저가 불가침 지정한 것만 propose.

## 집행 방식: 원본 1-read → 1-apply (덧대기 금지)

**규칙을 원본에 무조건 순차 적용하고 나중에 오버슈트를 패치하는 "덧대기" 는 금지다.** 그건 의도된 디자인까지 밀어버린 뒤 되돌리는 낭비다(R4 세리프 정체성을 통째로 날렸다가 복구한 실증). 대신 **편집 전에 원본을 한 번 통독해 5차원 타깃을 전부 뽑고, delta 만 한 번에 친다**:

1. **Phase A · 원본 측정 (편집 0)**: 5차원 현 상태를 계측한다 - 타이포는 **폰트 계열↔역할 맵**(어느 계열이 어느 역할을 담당하나), 그리드는 컬럼 fr↔공유 모듈 도출 여부, 색은 hue↔시맨틱 역할, 간격은 값↔그루핑, 오버라인은 타이틀과의 중복.
2. **Phase B · 단일 기계 판정 "역할 고정(의도) vs 랜덤(노이즈)"**: 각 차원을 **하나의 측정**으로 가른다 - 패턴이 역할에 고정돼 있으면(의도) 유지, 랜덤이면(노이즈) 고친다. 이게 blunt 를 유지하면서 의도를 살리는 핵심이다. "무조건 변환" 이 아니라 "역할 고정이면 유지, 랜덤이면 변환" - 조건은 단 하나의 측정값이라 여전히 기계적이다.
3. **Phase C · 타깃 1회 확정 → delta 만 1회 적용**: 되돌릴 일을 만들지 않는다.

## 5대 직설 규칙 (AI 통계적 실패 → 기계적 강제)

### 규칙 1 · 간격: base 는 대상에서 도출, 배수는 고정 → 위계를 계단으로, 과감히 넓힌다
- **AI 통계**: 산출물은 거의 항상 오밀조밀 과밀이고 **텍스트·UI 위계 그루핑이 없다**. 대표 증상 - overline·title·body 사이 간격이 **전부 동일**해서 다 한 덩어리로 뭉친다(블롭).
- **기계 규칙 (고정 px 절대 금지 - 이게 이 규칙의 핵심)**:
  - **(1) base unit 은 대상의 성격에서 도출한다 (측정, 고정값 아님)**. Phase A 의 읽기모드·콘텐츠 밀도로 판정 - **에디토리얼/매거진/롱폼 = 큰 base(24~32)**, 마케팅/랜딩 = 중(16~20), 조밀 앱/대시보드/데이터 = 작은 base(8~12). 같은 사다리라도 base 가 프로젝트 성격을 담는다.
  - **(2) 배수(tier 비율)는 고정이다**. 모든 간격 = `base × 고정배수` 로만 도출하고 요소마다 눈대중 px 금지:
    - hug (오버라인→타이틀, 한 유닛으로 밀착) = **0.25× base**
    - within (타이틀→본문, 그룹 내) = **1× base**
    - action (본문→CTA·링크) = **2× base**
    - component (블록↔블록, 카드↔카드) = **4× base**
    - section (섹션↔섹션) = **8× base**
    - hug:within = 1:4 라 오버라인이 타이틀에 확실히 붙고 그룹이 계단으로 읽힌다. 각 상위 티어는 하위의 2× 라 지각 가능.
  - **(3) 과감히**: base 를 후하게 잡는다. 특히 에디토리얼은 크게 - 대중이 "여백이 넉넉하고 위계가 보인다" 고 느낄 때까지. AI 과밀을 기본 가정으로 넓히는 쪽.
- **실증 교훈**: 고정 px(4/20/36 을 대상과 무관하게 떨굼)는 measure-first 위반이고 블롭을 못 푼다. base 를 대상에서 재고, 전 간격을 그 base 의 정수배로 재도출해야 "이 디자인의 리듬" 이 된다.
- **폐기된 옛 계약**: "안 답답하면 skip" 폐기. AI 산출물은 **기본 과밀로 간주**하고 항상 넓힌다 (유저가 "이미 넉넉하다" 하면 멈춘다).

### 규칙 2 · 관통 그리드: 컬럼은 공유 모듈에서만 나온다 → 랜덤 비정렬을 죽인다
- **AI 통계**: content-fit 로 열 폭이 섹션마다 제각각(.88/.92/.76fr 랜덤)이라 세로 분할선이 **의미없이 흩어지고**, content-fit 행 높이 때문에 가로선도 랜덤이라 세로선이 이웃 가로선을 **어중간하게 가로지른다**.
- **기계 규칙**: 모든 컬럼 경계를 **하나의 공유 모듈의 하모닉 분할**에서만 도출한다 (½·⅓·¼). **무조건 1자 관통이 아니다** - 상단이 1:1이면 하단은 0.5:0.5:1 이나 1:0.5:0.5 같은 **모듈의 응용**은 허용(오히려 권장). 죽이는 건 모듈에서 안 나오는 **랜덤 비정렬**뿐. 세로선은 이웃 가로선과 **코너/T-교차에서만** 만나고 어중간하게 관통하지 않는다. content-fit 랜덤 행은 균등 행(`repeat(N,1fr)`)이나 모듈 배수로 정렬.
- **정지**: 유저가 특정 섹션을 직접 다르게 잡아달라 하기 전까지 이 정렬은 기본 강제다.

### 규칙 3 · 중복 오버라인: 타이틀과 겹치면 무조건 삭제
- **AI 통계**: 모든 헤드라인 위에 반사적으로 오버라인(kicker·eyebrow·번호)을 깐다. 최악은 **타이틀과 의미가 동일한 오버라인** (`01 / Product` 위에 `Product`, `READ BY INTEREST` 위에 "Read by interest").
- **기계 규칙**: 오버라인이 바로 아래 타이틀과 **의미 중복이면 무조건 삭제** (직무유기). 오버라인이 **진짜 추가 정보**(타이틀에 없는 카테고리·순서 기능)를 줄 때만 남긴다. "모든 헤드라인에 오버라인" 패턴 자체를 깬다.

### 규칙 4 · 타이포: 이탤릭 무조건 제거 + 폰트는 역할고정이면 유지·랜덤이면 통일
- **AI 통계**: 헤드라인 한 단어만 세리프 이탤릭으로 갈아끼우거나, 폰트 계열을 **역할 없이 3계열+ 로 무질서 혼용**한다.
- **기계 규칙 (Phase B 역할고정 판정 적용)**:
  - (a) **이탤릭 무조건 제거** - `font-style: italic` 전부 삭제(판정 무관, 예외 없음, 장식 이탤릭 0).
  - (b) 폰트 계열은 **원본 read 의 계열↔역할 맵으로 판정** - 판정 신호는 **계열 수가 아니라 역할 고정 여부(bijection)** 다. 각 계열이 **한 역할에만** 쓰이고 각 역할에 **한 계열만** 매핑되면(예: serif = 전 디스플레이 / mono = 전 라벨 / sans = 전 본문) **정연한 시스템이므로 유지** - 3계열이어도 역할 고정이면 의도된 시스템이다(에디토리얼 정석). **통일은 오직**: 한 계열이 여러 역할을 랜덤하게 넘나들거나(예: 세리프가 어떤 본문엔 쓰이고 어떤 덴 안 쓰임), 한 역할에 여러 계열이 섞일 때(예: 헤드라인이 곳에 따라 serif/sans 랜덤). 위계는 폰트 교체가 아니라 **크기·굵기·간격**으로만.
  - **오버슈트 금지**: "무조건 한 패밀리 통일" 도, "계열 3개 이상이면 통일" 도 금지다 - 둘 다 역할 고정된 의도된 시스템을 죽인다(R4 실증: serif=디스플레이·mono=라벨·sans=본문 3계열 역할고정을 통째로 밀어 정체성 상실). 통일 트리거는 역할 랜덤일 때 하나뿐이다.

### 규칙 5 · 색: 시맨틱 먼저, 접근성으로 재분배
- **AI 통계**: 형제·정성 콘텐츠(카테고리·특성 나열)마다 유채색을 뿌려 hue 가 역할과 무관하게 난립하고, 대비(접근성)를 안 본다.
- **기계 규칙**: (a) **시맨틱 역할을 먼저 개념 정의** - 브랜드 + CTA + 실재하는 상태(오류/경고/성공)만 색 역할로 인정. (b) 역할 외 전부 중립 톤 스케일로 강등. (c) **접근성(대비) 기준으로 색 재분배** - 확정된 역할 색이 배경 대비 WCAG 통과하도록 명도 재배치. 색은 역할의 함수지 장식이 아니다.

## 공통 계약 (5규칙 위에)

- **문답 0.** 애매하면 묻지 말고 propose 로. 물어야 할 만큼 판단이 갈리면 그건 slop-fix 일이다.
- **의미 불가침.** 카피·문구·정보 구조·섹션 경계·순서는 자동 수정 대상이 아니다 (규칙 3 의 오버라인 삭제는 중복 장식 제거지 콘텐츠 삭제가 아니다). 콘텐츠 애매성은 propose.
- **되돌릴 수 있게.** 시작 전 git 확인, 더러우면 브랜치. **수정 1건 = 커밋 1개.** 버전관리 밖이면 `<대상>-quick` 복사 후 진행.
- **불가침 존중.** 유저가 "이건 두라" 한 시그니처(브랜드 장치·의도된 미학)는 규칙 적용에서 제외하고 propose 로만.
- **처방 출처 = SSOT, 런타임 로드.** 무엇이 슬롭이고 무엇으로 탈출하는지는 `src/data/aiSlopTaxonomyData.js` 의 tell/escape 에서 온다. 표현은 대상 프로젝트 토큰·관례로.
- em-dash(U+2014) 0건 (SSOT 원문 인용부 예외).

## 절차 (단일 패스)

0. **준비**: 대상·스택 파악, git 안전 확보. 과감도 기본값 = "5규칙 기계 적용".
1. **Phase A · 원본 통독 (편집 0)**: `node scripts/scan-slop-signals.mjs <대상> --json` + 렌더 계측. **5규칙 전 차원을 원본에서 한 번에 측정** - 계열↔역할 맵, 컬럼 fr↔모듈, hue↔시맨틱, 간격↔그루핑, 오버라인↔중복. 이 단계에서 편집하지 않는다.
2. **Phase B·C · 타깃 확정 후 delta 1회 적용 (5규칙 순서, 커밋 단위)**: 각 규칙마다 역할고정(의도)/랜덤(노이즈) 판정으로 타깃을 확정하고 delta 만 친다. 규칙1 간격 → 규칙2 그리드 → 규칙3 오버라인 → 규칙4 타이포 → 규칙5 색. 상류(구조·간격)부터, 과감한 diff. 삭제는 orphan-ref(미사용 var·클래스·키프레임)까지. **덧대기(무조건 변환 후 패치) 금지.**
3. **검증 (렌더 계측, 캐시 함정 주의)**: 수정 후 재계측 전 **반드시 캐시버스트**(새 포트나 `?v=` 로 애셋 강제 재요청) - 같은 URL 재로드는 옛 CSS 를 계산해 "수정이 안 먹혔다" 오판을 부른다(실측된 함정). 간격 차등비·분할선 모듈 도출·이탤릭 0·타입 패밀리 1개·색 역할 수를 계측으로 확인.
4. **리포트 + 2차 제안**:
```
## Slop Quick: <대상> <날짜>
### 적용된 5규칙 (커밋별)
| 규칙 | 적용 내용 | 계측 결과 | 커밋 |
### 체감 변화 (before/after 1쌍)
### 2차 제안 (애매·불가침 - 자동 안 함)
| 항목 | 왜 자동 불가 | 권장 |
```

## 흔한 함정 (이 스킬 전용)
- **소심하게 고치지 말 것.** 이 스킬의 실패 1위는 "몰 고쳤는지 모르겠다" 다. 규칙1 간격은 눈에 띄게 넓히고, 규칙2 그리드는 랜덤을 확실히 죽인다. 미세 조정은 이 스킬의 목적이 아니다.
- **그럴듯한 논리로 도망가지 말 것.** 다단계 판단 트리를 세우는 순간 실행이 왜곡된다 - 5규칙을 기계적으로 집행한다.
