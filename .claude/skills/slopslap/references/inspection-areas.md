# slopslap 영역별 점검·집행 규칙 (SSOT)

각 점검자/집행자는 **자기 영역 절만** Read 한다 (전체를 한 컨텍스트가 들지 않는다 = 용량 드롭 방지). 무엇이 슬롭이고 무엇으로 탈출하는지의 원천은 `${CLAUDE_PLUGIN_ROOT}/src/data/aiSlopTaxonomyData.js` 의 tell/escape 이며, 아래는 그 위에 얹는 영역별 판정·집행 계약이다.

공통 원칙: **"AI 산출물이 통계적으로 이렇게 망친다 → 기계적으로 이렇게 강제한다"**. 정성 판단이 아니라 단일 측정 트리거로 가른다. 값은 고정 px 가 아니라 대상에서 도출한 단위 × 고정 배수로.

## 영역 A · 오버라인·중복 텍스트 (집행 순서 1)

- **AI 통계**: 모든 헤드라인 위에 반사적으로 오버라인(kicker·eyebrow·번호)을 깐다. 최악은 타이틀과 의미 동일한 오버라인(`01 / Product` 위에 `Product`).
- **명시적 삭제 리스트 (판단 아님 - 아래 패턴에 걸리면 무조건 삭제. 각 오버라인을 이 리스트에 전수 대조)**:
  1. 오버라인 텍스트가 바로 아래 타이틀을 **재진술하거나 부분집합**(의미 중복).
  2. **반복되지 않는(형제 ≤1) 단발 섹션·헤딩** 위의 topic·kicker 라벨.
  3. 기능 앵커(목차·스텝·페이지네이션 링크) **없는 순수 번호·인덱스**.
  4. all-caps eyebrow·pill eyebrow 배지가 위 1~3 중 하나에 해당.
  - **유지 조건 (이것만 예외로 남긴다)**: **≥2개 반복 형제 블록**에 붙어 있고 **+** 각 타이틀에 없는 카테고리/순번 값을 담을 때만.
  - "시리즈냐"를 주관 판단하지 말고, 위 삭제 리스트 1~4 에 걸리는지 기계적으로 대조한다.
- **간격 (영역 C hug 연동)**: 유지하는 오버라인은 타이틀과의 간격이 **타이틀 line-height 개행 간격보다 커야** 한다 (안 그러면 타이틀 한 줄로 오독). findings 에 이 값 기록.
- SSOT: `numbered-overline-fetish`·`pill-eyebrow-badge`·`all-caps-eyebrow`.
- 집행 = 마크업에서 해당 오버라인 노드 + orphan CSS 삭제. 카피·콘텐츠 본문은 불가침(중복 장식만 삭제).

## 영역 B · 레이아웃·컨테이너·이미지 (집행 순서 2)

순서가 곧 결정트리 - 위에서 정하면 아래가 따라온다.

1. **타입 먼저 (성격 판정)**: 스토리텔링·USP·쇼케이스 랜딩 + 컨셉추얼 히어로 → **fluid, 전체 뷰포트 n빵**(히어로·씬 100vh, 뷰포트 스케일 여유). 조밀 앱·데이터·대시보드 → fixed/contained. **fluid ≠ 콘텐츠 팽창**: 셀 안 콘텐츠 크기는 그대로, 정렬만. **100vh 는 선언이 아니라 측정**: 콘텐츠 높이 합(폰트·라인수·패딩·gap 합산 또는 렌더 실측)이 뷰포트에 맞는지 재고, 넘치면 뷰포트 씬 아님 → 내부 간격·크기 축소 또는 100vh 해제. 이미지 natural 크기가 씬을 넘기는 최빈 원인이니 캡. SSOT: `layout-type-misfit`·`undecided-layout-type`.
   - **(1b) BOLD 게이트 — 투박·저밀도로 스타일이 사는 케이스인가 (상류 단일 판단, 엣지케이스)**: 이 판정은 **파이프라인에서 한 번만**(성격 판정 단계) 하고, 결과를 `BOLD=on/off` 플래그로 하류 영역(특히 C)에 전달한다. **하류는 재판정 금지 — 플래그만 소비**(판단 지점 1개로 격리 = 컨텍스트 부하·불일치 방지). **판정은 AND, 둘 다 참일 때만 `BOLD=on`**:
     - (a) **투박한 스타일 어휘 ≥3종 반복**: ① 두꺼운 solid border(≥3px 진한/검정), ② hard box-shadow(offset ≥4px + **blur 0**), ③ flat 고채도 원색 fill, ④ 초대형 볼드 타이포(헤드라인 특대·900), ⑤ radius 0/극단.
     - (b) **저밀도로 스타일이 사는 콘텐츠인가 — 느슨하게, on 쪽으로 (관대)**: **AI 는 여백을 못 견뎌 불필요하게 고밀도로 채운다** — 그러니 **겉보기 빽빽함으로 off 하지 마라**. `off`(진짜 밀도)는 **저밀도로는 못 만드는, 기능적으로 밀도가 필수인 것일 때만**: 데이터 표·대시보드·실데이터 긴 목록·폼/빌더/세그·설정 패널 등 **인터랙티브·데이터 밀집**. 그 외 — 카드 그리드·텍스트 블록 여럿·3열 피처 카드 등 **겉보기 밀도는 저밀도로 간주**(AI 가 불필요하게 채운 것이지 콘텐츠가 요구한 게 아님). **면적 기준**: 페이지 면적의 상당부를 히어로·스테이트먼트·여백-주역 섹션이 차지하면 저밀도. **경계면 on**.
     - **(a) 어휘는 여전히 필수 조건**: (a) 불충족(투박한 스타일 어휘 없음)이면 (b) 와 무관하게 `off` — 스타일이 없는 화면을 억지로 크게 만들지 않는다. 단 (a) 가 있으면 (b) 는 위처럼 관대하게 on 쪽. (STACKBOX void 는 (b) 판정이 아니라 "간격 무차별 2배"가 원인이었음 → 그건 C 1-neo·B 스케일 규칙에서 이미 정비: on 이어도 요소·표면 위주로 키우고 인접 gap·죽은 void 가드 지킨다.)
     - **`BOLD=on` → 과감히 키운다**: UI·버튼·아이콘 크기, border·hard-shadow, 표면 내부 패딩, **그리고 간격·패딩까지**(저밀도라 큰 여백이 곧 스타일, void 아님). 인접 요소는 **gap ≥ hard-shadow offset**(그림자 겹침 방지). 여전히 도출값(눈대중 아님)·초점 고립 유지. 간격 스케일업은 영역 C 가 이 플래그 보고 집행.
     - **`BOLD=off` → 전 영역 기존 규칙대로**(키우지 않음). SSOT: `layout-type-misfit`.
2. **모듈**: 컬럼 경계는 **공유 모듈의 하모닉 분할**(½·⅓·¼)에서만. 무조건 1자 관통 아님(상단 1:1→하단 0.5:0.5:1 응용 OK), 죽이는 건 랜덤 비정렬뿐. content-fit 랜덤 행은 균등 행(`repeat(N,1fr)`)으로. SSOT: `undisciplined-grid`.
   - **(2a) 폭도 도출한다 (fixed·content-fit 금지)**: 컨테이너·컬럼·셀 폭이 **공유 모듈(하모닉 분할·max-width 토큰)에서 도출**되지 않고 제각각 **고정 px** 나 **내용맞춤(content-fit)** 으로 박혀 섹션마다 폭이 어긋나면 = 균형 파괴 슬롭. 간격과 똑같이 **폭도 단일 척도에서 도출**한다. 기계 트리거: 형제/인접 섹션의 콘텐츠 컨테이너 max-width·컬럼 폭이 **서로 다른 값**(예 900 / 1120 / auto 혼재)이면 하나의 페이지 measure 토큰(또는 그 하모닉 배수)으로 통일. 셀 폭을 내용길이에 맡기지 말고 그리드 트랙(`1fr`·하모닉)으로 강제.
   - **(2b) content measure ↔ 정보밀도 정합 (과협소 measure 금지)**: AI 통계 = 좌우 여백은 후하게 비우면서 **콘텐츠 폭(measure)을 정보밀도 대비 과하게 좁게** 잡아, 여백만 넓고 읽는 영역이 옹색해짐(카드 다열·피처 그리드·표가 좁은 컬럼에 짓눌림). **여백은 프레임이지 콘텐츠를 압착하는 게 아니다(폭 버전).** 기계 트리거: 콘텐츠 컨테이너 max-width 가 **담긴 밀도가 요구하는 폭보다 좁으면**(예: 3~4열 카드/피처 그리드인데 measure ≤ 900, 또는 뷰포트 대비 콘텐츠 점유율이 과소해 좌우 여백이 콘텐츠보다 넓음) measure 를 밀도에 맞춰 **넓힌다**. 산문 단일 컬럼(45~75em 가독 measure)은 예외 — 좁은 게 정답. 밀도(열 수·요소 수)로 판정하지 화면 크기로만 판정 금지. SSOT: `undisciplined-grid`(폭 미도출)·`layout-type-misfit`(밀도-폭 불일치).
   - **(2c) 시리즈 UI(배열 매핑) 정렬**: 콘텐츠 상수화(0단계)가 식별한 **반복 시리즈 배열**(리스트·카드·스텝·요금티어·기능항목)은 각 항목의 **불릿·번호·아이콘을 첫 줄 텍스트 baseline 에 고정**한다 — **세로 중앙(`top:50%` / align-items center) 금지**(2줄+ 항목에서 불릿이 첫 줄과 어긋나 시리즈가 들쭉날쭉; 마커는 `top` 을 첫 줄 line-height 중앙 ≈ `0.5lh` 에 고정하거나 flex `align-items:baseline`/`flex-start`). 항목 규격(마커 크기·들여쓰기·번호 자리수) **동일 정렬**, 넘버링은 자리수 맞춤·우측정렬. 기계 트리거: 시리즈 li/카드 마커가 `top:50%`·`align center` 인데 항목 텍스트가 가변 줄수면 위반. SSOT: `undisciplined-grid`.
3. **최소 컨테이너** — 아래는 **판단이 아니라 기계 대조 리스트**다. 걸리면 무조건 집행:
   - **(3a) box-in-box = 무조건 flatten**: 컨테이너가 (border / 배경-fill / box-shadow / radius 중 하나라도) 가진 채로, 같은 것 중 하나라도 가진 **자식을 직접 감싸면** = 표면 중복. **역할이 있든 없든** 안쪽 표면을 벗겨 자식을 **상단선·구분선 + 여백**으로 바꾸고, 한 그룹에 표면은 **최대 1겹**만 남긴다. (엄격 'ghost wrapper' 정의로 통과시키던 실패를 이 기계 규칙이 대체 - 표면이 표면을 감싸면 무조건 걸린다. 예: 패널(bg+border)이 옵션박스(bg+border)들을 감싸면 옵션박스 표면 제거 → borderline 행.)
   - **(3b) 유령 래퍼 삭제**: 레이아웃 기여 0(단일 자식 + grid/flex·여백·배경·보더·시맨틱 어느 것도 없음) 래퍼는 삭제·언랩.
   - **(3c) 좌측 컬러 보더 절대 금지**(대표적 AI slop): 필요해 보여도 이 스킬에선 안 쓰고 제거, 강조는 figure-ground(명도·크기·여백)로.
   - **(3d) 표면 색 통일**: 터미널·코드블록 등은 배경·가로스크롤바(`scrollbar-color` + webkit)까지 한 색으로.
   - **(3e) 컨테이너 남발 금지 (표면은 꼭 필요할 때만)**: 표면(bg/border/radius/shadow)은 **정보 그룹을 실제로 구분해야 할 때만** 쓴다. 단순 리스트 항목·행·단문을 **각각 박스로 감싸지 마라** — 불릿+텍스트 행, 상단 구분선(hairline), 여백으로 충분하다. 특히 콘텐츠 상수화가 잡은 **시리즈 항목을 개별 카드화**하는 건, 그 항목이 독립적으로 밀도 있는 정보 단위(제목+본문+메타+액션을 다 가진)가 **아닌 한 남발** → 표면 벗기고 구분선 행으로. 기계 트리거: 시리즈 배열의 각 항목이 표면(bg/border/radius)을 갖는데 항목 내용이 단문/1~2필드면 위반. (3a box-in-box 가 '표면 속 표면'을 친다면, 3e 는 '**애초에 표면이 필요 없는 것**'을 친다.)
   - SSOT: `meaningless-container-nesting`·`colored-left-border-cards`. 각 항목을 findings 에 "해당/무해당"으로 전수 기록(조용한 누락 차단).
4. **이미지 가드**: natural 초과 렌더 금지(업스케일=0). 시리즈/반복 포맷에 안 맞는 이미지는 늘리거나 여백으로 때우지 말고 **홀수 1개 삭제**(리덕티브).

## 영역 C · 간격 (집행 순서 3)

- **AI 통계**: 거의 항상 과밀 + 위계 그루핑 부재(overline=title=body 간격 동일 = 블롭).
- **(1) base unit 은 대상 성격에서 도출** (측정, 고정 px 아님): 에디토리얼/매거진/스토리텔링·USP 랜딩 = 큰 base(24~32), 일반 정보·마케팅 랜딩 = 중상(20~24), **조밀 앱/대시보드/데이터만** 작은 base(8~14). 기본은 관대 쪽.
- **(1-neo) BOLD 플래그 연동 (판정은 영역 B 1b 상류에서 이미 완료 — 여기서 재판정 금지, 플래그만 소비)**: 지휘자가 전달한 `BOLD=on` 이면(투박·저밀도로 스타일이 사는 케이스로 이미 판정됨) 이 화면은 **간격·패딩을 과감히 ↑** — 도출 base 를 넉넉히 잡고 사다리를 상향한다(저밀도라 큰 여백이 곧 스타일, void 아님). `BOLD=off` 면 이 항목 무시하고 (1) 통상 base 도출대로. **왜 판정을 여기서 안 하나**: 밀도 있는 화면에서 간격만 키우면 죽은 void(STACKBOX 실패)라 "밀도 판단"이 필수인데, 그 판단은 상류 AND 게이트(B 1b) 한 곳에서만 하고 C 는 플래그만 본다(판단 중복·불일치 방지).
  - **hard-shadow 겹침 방지**: hard-shadow 쓰는 **인접** 요소(나란한 버튼 등)는 gap ≥ shadow offset + 여유. shadow 키웠으면 gap 도 그만큼(안 그러면 그림자가 옆 요소에 부딪혀 지저분).
  - 초점 고립 유지(그룹 내부 조임 유지). base 도출·사다리 원칙은 그대로 쓰되 **네오브루탈이라고 base 를 2배 하지 않는다** — 밀도는 콘텐츠가 정한다. 무드 아니면 무시.
- **(2) 배수(tier)는 고정**, 모든 간격 = `base × 고정배수`:
  - hug(오버라인→타이틀) = 0.25× base, **단 타이틀 line-height 개행 간격보다 크게(floor)** - 작으면 개행 간격 위로.
  - within(타이틀→본문) = 1× / action(본문→CTA) = 2× / component(블록↔블록) = 4× / section(섹션↔섹션) = 8× base.
- **(3) 과감히**: base 를 후하게. AI 과밀을 기본 가정으로 넓힌다.
- **(4) 초점 고립 (게슈탈트 1번)**: 여백은 **응집된 초점 그룹을 감싸는 프레임**이지 그룹을 **분해하는 쐐기가 아니다**. 히어로(eyebrow+헤드라인+설명+CTA)는 내부를 hug/within 으로 **tight 한 덩어리**로 두고, 그 **바깥 마진**만 후하게 비워 고립. **금지: 초점 요소들 사이 간격을 균일 팽창시켜 그룹을 흩뜨리는 것**(성김). **죽은 void 금지**: CTA 아래·섹션 사이 목적 없는 거대 공백은 고립이 아니라 결함 - 섹션 간격은 후하되 진공 금지, "완성돼 보이는가" 확인. 텍스트 몰린 구간은 gutter·행 간격 확장.
- SSOT: `unscaled-spacing-ladder`·`unpartitioned-space`. 구속: 여백 눈대중 금지, 반드시 선언된 사다리에서 도출.

## 영역 D · 타이포 (집행 순서 4)

- **AI 통계**: 헤드라인 한 단어만 세리프 이탤릭으로 갈거나, 폰트 계열을 역할 없이 무질서 혼용.
- **(a) 이탤릭 무조건 제거**: `font-style: italic` 전부 삭제(예외 없음).
- **(b) 폰트는 계열↔역할 맵으로 판정** - 신호는 계열 수가 아니라 **역할 고정(bijection)**: 각 계열이 한 역할에만/각 역할에 한 계열만(serif=전 디스플레이 / mono=전 라벨 / sans=전 본문)이면 **정연한 시스템이므로 유지**(3계열이어도). **통일은 오직** 한 계열이 여러 역할을 랜덤하게 넘나들거나 한 역할에 여러 계열이 섞일 때. 위계는 폰트 교체가 아니라 크기·굵기·간격으로만.
- **오버슈트 금지**: "무조건 한 패밀리 통일"도 "3계열+면 통일"도 금지 - 역할 고정된 의도된 시스템을 죽인다.
- SSOT: `italic-serif-accent`·`monospace-body-aesthetic`.

## 영역 E · 색 (집행 순서 5)

- **AI 통계**: 형제·정성 콘텐츠마다 유채색 살포, 대비(접근성) 무시.
- **(a) 시맨틱 역할 먼저 정의**: 브랜드 + CTA + 실재하는 상태(오류/경고/성공)만 색 역할로 인정.
- **(b) 역할 외 전부 중립 톤 스케일로 강등.**
- **(c) 접근성(대비) 기준 재분배**: 확정 역할 색이 배경 대비 WCAG(본문 4.5:1) 통과하도록 명도 재배치.
- 집행 시 orphan 색 변수·클래스 스윕. SSOT: `decorative-semantic-color`·`rainbow-status-list`·`iridescent-palette`.

## findings 스키마 (모든 영역 공통)

각 점검자는 `findings-<영역>.md` 에 항목별로:

```
- id: <SSOT id 또는 area-slug>
  문제: <한 줄>
  근거: <정적 측정값 - 파일:줄 또는 계산값(대비 x:1, 분할선 x%, 폰트 역할맵 등)>
  처방: <기계적 수정 - 무엇을 어떤 값으로. base 도출값·배수 명시>
  검증(check): <소스에서 참/거짓으로 실측 판정할 술어 - "충족" 조건을 grep·값 비교로 확정할 수 있게. 예: "grep 'scrollbar-color' 가 .code pre 블록에 있고 값이 --code-bg/border" / "hero max-width 가 var(--measure) 참조" / "font-style:italic 매치 0건". 상태(완료/미완료)를 텍스트로 적지 말 것 — check 는 매번 소스에서 계산된다.>
  집행순서: A|B|C|D|E
```

**점검표 = 평가 함수 (상태 주장 금지)**: findings 항목은 "완료/무해당" 같은 **상태를 선언하지 않는다**. 각 항목은 `검증(check)` 술어를 갖고, 집행자·재점검자는 그 술어를 **소스에서 매번 실측**해 참(이미 충족 → skip)/거짓(집행)으로 가른다. **"완료" 라고 적힌 텍스트를 신뢰하지 마라** — 옛 findings 를 재사용하거나 대상 상태가 바뀌면 그 기록이 어긋난다(터미널 스크롤바색 누락이 이 실패). 집행자는 check=거짓인 항목만 tick 하며 처방 적용. 재점검자는 같은 check 를 소스/렌더에 재실측해 참/거짓 기록(주관 판정 아님). **이 평가·집행은 서브에이전트가 수행하고, 지휘자(메인 세션)는 직접 판단·수정하지 않는다**(오케스트레이션만).
